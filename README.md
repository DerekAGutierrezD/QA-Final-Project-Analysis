# QA Comprehensive Final Project: Functional & Technical Analysis

## Descripción del Proyecto
Este repositorio contiene el entregable final del programa de QA Engineering. El proyecto demuestra habilidades integrales en el ciclo de vida de desarrollo de software (SDLC), desde la teoría de redes y bases de datos hasta la ejecución práctica de pruebas funcionales y reporte de errores en sistemas reales.

## Áreas de Competencia Demostradas

### 1. Fundamentos Técnicos (Teoría)
El proyecto incluye un análisis profundo sobre:
*   **Arquitectura de Software:** Comportamiento cliente-servidor y procesos DNS/HTTP.
*   **Protocolos:** Manejo de HTTP/HTTPS, métodos (GET, POST, PUT, DELETE) y códigos de respuesta (2xx, 4xx, 5xx).
*   **Gestión de Datos:** Diferencias entre bases de datos relacionales y NoSQL, con ejecución de consultas **SQL (JOINs, GROUP BY, filtrado)** para validación de lógica de negocio.

### 2. Diseño de Pruebas y QA Funcional
Aplicación de técnicas de caja negra para garantizar la cobertura de requisitos:
*   **Análisis de Clases de Equivalencia y Valores Límite:** Aplicado rigurosamente a campos de entrada para maximizar la detección de errores con el mínimo número de casos.
*   **Tablas de Decisiones:** Utilizadas para optimizar la lógica de formularios complejos.
*   **Listas de Chequeo vs. Casos de Prueba:** Diseño y ejecución de una suite completa de pruebas funcionales.

### 3. Reporte y Calidad (Execution)
*   **Ejecución de Pruebas:** Evaluación de flujos de usuario (alquiler de scooters, estado de pedidos, validaciones de interfaz).
*   **Gestión de Defectos:** Reporte estructurado de errores encontrados, incluyendo su criticidad y trazabilidad.

## Logros Destacados
- **Análisis de Datos:** Ejecución de queries complejas en SQL para resolver inconsistencias de negocio (gerencia, salarios, comportamiento de usuarios).
- **Rigor en Pruebas:** Implementación de estrategias de partición de equivalencia y valores límite, identificando errores de diseño en los límites de los formularios.
- **Mentalidad de QA:** Entendimiento claro del valor de la automatización vs. pruebas manuales, priorizando la estabilidad del proyecto y el retorno de inversión (ROI) en las pruebas.
